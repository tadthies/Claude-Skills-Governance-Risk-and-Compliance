WCAG 2.1 AA Contrast Analysis
==============================

WCAG 2.1 AA requires a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text
(>=18pt regular or >=14pt bold) and UI components. Placeholder text and disabled controls have
special considerations noted below.

RESULTS BY COMBINATION
-----------------------

1. Body text: #767676 on #FFFFFF
   Contrast ratio: 4.54:1
   Result: PASS (barely meets the 4.5:1 threshold for normal text)
   Action: No change required, but consider darkening slightly for extra margin.

2. Navigation links: #0066CC on #FFFFFF
   Contrast ratio: 5.57:1
   Result: PASS
   Action: No change required.

3. Button text: #FFFFFF on #3D9970 (green button)
   Contrast ratio: 3.51:1
   Result: FAIL for normal text (requires 4.5:1). Passes only if button label is large text (>=18pt).
   Action: Darken the green background. Suggested replacement: #2D7A55 (ratio ~5.22:1) or
   #1E6B47 (ratio ~6.46:1) to comfortably pass for all text sizes. Keep white button text.

4. Placeholder text in inputs: #AAAAAA on #FFFFFF
   Contrast ratio: 2.32:1
   Result: FAIL (well below the 4.5:1 requirement)
   Note: WCAG 2.1 does not exempt placeholder text from contrast requirements. It must meet
   the same 4.5:1 ratio as regular text (Success Criterion 1.4.3).
   Action: Darken the placeholder text significantly. Suggested replacements:
   - #767676 on #FFFFFF => 4.54:1 (just passes)
   - #6B6B6B on #FFFFFF => 5.33:1 (comfortable pass)
   - #595959 on #FFFFFF => 7.00:1 (strong pass, preferred)

5. Error messages: #CC0000 on #FFFFFF
   Contrast ratio: 5.89:1
   Result: PASS
   Action: No change required.

6. Disabled button: #999999 on #CCCCCC
   Contrast ratio: 1.77:1
   Result: FAIL — however, WCAG 2.1 explicitly exempts disabled UI components from contrast
   requirements (Success Criterion 1.4.3, exception clause). Disabled controls that are
   "inactive" do not need to meet contrast thresholds.
   Action: No contrast fix is technically required under WCAG 2.1 AA. That said, very low
   contrast on disabled buttons can confuse users who are uncertain why they cannot interact
   with the control. Best practice is to provide a visible disabled state and supplement with
   a clear disabled indicator (e.g., greyed appearance + cursor:not-allowed + aria-disabled).

SUMMARY OF FAILURES REQUIRING REMEDIATION
------------------------------------------

| Element            | Foreground | Background | Ratio  | Status | Suggested Fix                        |
|--------------------|-----------|-----------|--------|--------|--------------------------------------|
| Button text        | #FFFFFF   | #3D9970   | 3.51:1 | FAIL   | Change bg to #2D7A55 or darker green |
| Placeholder text   | #AAAAAA   | #FFFFFF   | 2.32:1 | FAIL   | Change text to #767676 or darker     |

ELEMENTS THAT PASS
------------------

|--------------------|-----------|-----------|--------|--------|
| Body text          | #767676   | #FFFFFF   | 4.54:1 | PASS   |
| Navigation links   | #0066CC   | #FFFFFF   | 5.57:1 | PASS   |
| Error messages     | #CC0000   | #FFFFFF   | 5.89:1 | PASS   |
| Disabled button    | #999999   | #CCCCCC   | 1.77:1 | EXEMPT |

NOTES
-----
- Contrast ratios calculated using the WCAG relative luminance formula per WCAG 2.1 SC 1.4.3.
- "Large text" is defined as at least 18pt (24px) regular weight or 14pt (approximately 18.67px) bold.
- The disabled button exemption applies only when the component is genuinely non-interactive
  (aria-disabled="true" or the HTML disabled attribute is present).
- WCAG 2.1 AA compliance for colour contrast is assessed under Success Criterion 1.4.3 (Contrast Minimum).
- The higher WCAG 2.1 AAA standard requires 7:1 for normal text and 4.5:1 for large text.
